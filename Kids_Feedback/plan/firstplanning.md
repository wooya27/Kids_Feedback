# 줄넘기 수업 기록 기반 아이별 맞춤 성장 피드백 시스템

> AX/DX 엔지니어 포트폴리오용 2주 MVP 기획 문서
> 

## 0. 프로젝트 개요

**프로젝트 이름:** 줄넘기 수업 기록 기반 아이별 맞춤 성장 피드백 자동 생성 시스템

**한 줄 설명:** 줄넘기 수업에서 아이별 기록, 정서 반응, 어려워한 점, 아이 특징 메모를 누적하고, LLM과 RAG를 활용해 다음 수업 목표와 학부모용 피드백을 자동 생성하는 시스템입니다.

```
현장 경험에서 문제 발견
→ 아이별 특징과 수업 기록을 데이터화
→ 강사의 반복 피드백 작성 업무 자동화
→ LLM으로 맞춤 피드백 생성
→ RAG로 지도 기준 보정
→ 웹 대시보드로 성장 기록 확인
```

## 1. 프로젝트 배경

줄넘기 수업에서는 아이마다 필요한 지원 방식이 다릅니다. 어떤 아이는 줄넘기 개수보다 정서적 안정이 먼저 필요하고, 어떤 아이는 10개 목표보다 1개 성공 경험이 더 중요합니다.

강사나 보조강사가 모든 아이의 이름, 특성, 이전 기록, 정서 반응을 계속 기억하기는 어렵습니다. 또한 학부모에게 전달할 피드백은 부정적이지 않으면서도 구체적이어야 하기 때문에 작성 부담이 있습니다.

이 프로젝트는 아이별 특징과 수업 기록을 누적하고, 최근 성장 흐름과 지도 가이드를 참고하여 다음 수업 목표와 피드백을 자동 생성합니다.

## 2. 문제 정의

### 현장에서 발견한 문제

- 아이마다 필요한 지원 방식이 다릅니다.
- 아이 이름과 특징을 빠르게 기억하는 것이 수업 신뢰 형성에 중요합니다.
- 강사가 모든 아이의 이전 반응과 성향을 기억하기 어렵습니다.
- 수업 후 학부모용 피드백 작성에 시간이 듭니다.
- 줄넘기 개수만으로는 아이의 성장을 충분히 설명하기 어렵습니다.

### 기술로 바꾼 문제

```
아이별 수업 기록과 특징 메모를 구조화하고,
LLM과 RAG를 활용해 다음 수업 목표, 강사용 메모, 학부모용 피드백을 자동 생성한다.
```

## 3. 타겟 사용자

### 1차 사용자: 줄넘기 강사 / 보조강사

강사는 수업 전 아이 프로필 카드를 확인하고, 수업 후 기록을 입력합니다. 기록 항목은 줄넘기 개수뿐 아니라 어려워한 점, 정서 반응, 필요한 지원, 준비물 등을 포함합니다.

### 2차 사용자: 학부모

학부모는 아이가 단순히 몇 개를 했는지보다 어떤 부분에서 성장했는지 알고 싶어합니다. 시스템은 학부모가 이해하기 쉬운 따뜻한 문장으로 성장 피드백을 생성합니다.

## 4. 핵심 가치 제안

### 4.1 강사의 업무 부담 감소

수업 후 매번 문장을 새로 쓰는 부담을 줄입니다. 강사는 기록만 입력하고, AI가 만든 초안을 검토하면 됩니다.

### 4.2 아이별 맞춤 피드백

단순히 줄넘기 개수만 보는 것이 아니라, 아이의 성향, 정서 반응, 좋아하는 칭찬 방식, 성장 흐름을 함께 반영합니다.

### 4.3 피드백 품질 안정화

RAG를 통해 지도 원칙과 학부모 피드백 작성 규칙을 참고하도록 하여, 비교 표현이나 진단 표현을 줄입니다.

### 4.4 수업 준비 도구화

아이 프로필 카드를 통해 강사가 수업 전에 아이의 이름, 특징, 주의점, 선호하는 피드백 방식을 빠르게 확인할 수 있습니다.

### 개인화 반영

### 안전한 표현

나쁜 예시:

```
민준이는 집중력이 부족합니다.
민준이는 운동 능력이 낮습니다.
다른 아이들보다 많이 뒤처집니다.
```

좋은 예시:

```
민준이는 시작 타이밍을 익히는 과정에 있습니다.
작은 성공 경험을 통해 자신감을 쌓아가는 단계입니다.
```

### 목표의 현실성

다음 목표는 최근 기록보다 조금 높은 수준이어야 합니다. 예를 들어 오늘 2개 성공한 아이에게 다음 목표를 20개로 제안하면 부적절합니다.

### RAG 근거 반영

검색된 지도 가이드가 피드백에 자연스럽게 반영되어야 합니다.

## 8. 아이 프로필 카드 기능

### 기능 정의

아이 프로필 카드는 강사가 아이 이름과 특징을 빠르게 기억하고, 아이별 성향을 수업 준비와 피드백 생성에 활용할 수 있도록 돕는 기능입니다.

### 왜 중요한가?

보조강사 경험상 아이의 이름을 정확히 불러주고, 이전 수업에서 어려워했던 점을 기억해주는 것만으로도 아이의 수업 몰입도가 달라질 수 있습니다.

따라서 단순히 줄넘기 기록만 저장하는 것이 아니라, 아이별 외우기용 특징, 성향, 좋아하는 칭찬 방식, 주의할 점을 함께 저장해야 합니다.

### 프로필 카드 항목

```
이름
나이
수준
외우기용 특징
성향
좋아하는 칭찬 방식
주의할 점
수업 참여 스타일
준비물 특이사항
강사 메모
```

### 예시 데이터

```json
{
  "name": "민준",
  "age": 7,
  "level": "초급",
  "memory_hint": "파란 줄넘기를 들고 오고, 성공하면 양손을 들고 좋아함",
  "personality": "처음에는 조심스럽지만 칭찬을 받으면 적극적으로 도전함",
  "preferred_feedback": "개수보다 시도한 점을 칭찬해주면 좋음",
  "caution_note": "실패를 반복하면 금방 위축될 수 있음",
  "participation_style": "선생님과 함께하면 더 잘 따라옴",
  "material_note": "개인 줄넘기를 자주 깜빡함",
  "teacher_memo": "처음 목표는 낮게 잡고 성공 경험을 먼저 제공"
}
```

### 강사 UX 관점

```
민준 → 실패에 민감함 → 작은 성공 경험 먼저
서연 → 타이밍은 약하지만 집중력 좋음 → 박자 연습
지호 → 준비물 자주 빠뜨림 → 수업 전 체크 필요
```

이 기능은 단순 기록 기능이 아니라 수업 준비 도구입니다.

### LLM 개인화 관점

같은 줄넘기 3개라도 아이 특징에 따라 피드백이 달라져야 합니다.

```
아이 A: 실패에 민감함
→ 작은 성공을 쌓으며 자신감을 만들어가는 중입니다.

아이 B: 도전 욕구가 강함
→ 다음 수업에서는 연속 5개에 도전해볼 수 있습니다.
```

### 개인정보와 노출 정책

`memory_hint`는 강사용 정보입니다. 학부모용 피드백에는 직접 노출하지 않아야 합니다.

피해야 할 예시:

```
파란 줄넘기를 들고 오는 민준이는...
```

프롬프트 규칙:

```
외우기용 특징은 강사가 아이를 기억하기 위한 참고 정보로만 사용하고,
학부모용 피드백에는 직접 쓰지 마세요.
```
