## 22. 면접 답변

### Q1. 왜 RAG를 썼나요?

```
LLM만 사용하면 피드백 문장은 자연스럽게 만들 수 있지만, 지도 원칙이 일관되지 않을 수 있습니다.
예를 들어 실패에 민감한 아이에게는 개수보다 시도와 성공 경험을 칭찬해야 하는데,
이런 기준을 매번 프롬프트에 길게 넣기보다 지도 가이드를 문서화하고 RAG로 검색해 넣는 구조가 더 확장 가능하다고 판단했습니다.
```

### Q2. 왜 LangGraph나 멀티에이전트는 안 썼나요?

```
이번 MVP의 핵심은 복잡한 도구 호출이 아니라 아이별 기록 기반 피드백 생성입니다.
따라서 LangGraph 기반 멀티에이전트는 오히려 범위를 키울 수 있다고 판단했습니다.
먼저 FastAPI, PostgreSQL, LLM, RAG로 단일 흐름을 안정적으로 만들고,
향후 알림, 수업 계획 자동화, 선생님별 워크플로우가 추가되면 LangGraph로 확장할 수 있습니다.
```

### Q3. 아이 특징 메모는 왜 넣었나요?

```
보조강사 경험상 아이 이름을 정확히 기억하고, 이전 수업에서 어떤 반응을 보였는지 기억해주는 것이 수업 몰입도에 중요했습니다.
그래서 단순히 줄넘기 개수만 저장하는 것이 아니라, 아이를 기억하기 위한 특징, 좋아하는 칭찬 방식, 주의할 점을 함께 저장하는 프로필 카드 기능을 넣었습니다.

이 정보는 강사가 수업 전에 아이를 빠르게 파악하는 데 쓰이고, LLM 피드백 생성 시에는 개인화 컨텍스트로 사용됩니다.
다만 외우기용 특징이나 민감한 메모가 학부모용 피드백에 그대로 노출되지 않도록 프롬프트 규칙을 분리했습니다.
```

### Q4. 개인정보 문제는 어떻게 처리했나요?

```
MVP에서는 실제 아동 정보가 아니라 가명 샘플 데이터를 사용했습니다.
또한 생성 결과는 학부모에게 바로 전송하지 않고, 반드시 강사가 검토한 뒤 사용할 수 있도록 설계했습니다.
아이의 행동을 진단하거나 비교하는 표현은 금지 규칙으로 프롬프트에 포함했습니다.
```
