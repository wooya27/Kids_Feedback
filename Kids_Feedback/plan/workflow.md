

### 시나리오 4: 성장 기록 확인

강사는 아이별 과거 기록과 피드백 이력을 확인합니다. Streamlit 화면에서는 줄넘기 개수 변화 그래프를 보여줍니다.

### 사용자 플로우

```mermaid
flowchart TD
    A[강사 웹 접속] --> B[아이 선택]
    B --> C[아이 프로필 카드 확인]
    C --> D[오늘 수업 기록 입력]
    D --> E[기록 저장]
    E --> F[피드백 생성 버튼 클릭]
    F --> G[최근 기록 조회]
    G --> H[RAG 지도 가이드 검색]
    H --> I[LLM 프롬프트 구성]
    I --> J[피드백 생성]
    J --> K[강사 검토]
    K --> L[피드백 저장]
    L --> M[성장 기록 확인]
```

### AI 워크플로우

```mermaid
flowchart TD
    A[lesson_record_id 입력] --> B[아이 프로필 조회]
    B --> C[최근 수업 기록 5개 조회]
    C --> D[검색 쿼리 생성]
    D --> E[RAG 문서 검색]
    E --> F[프롬프트 생성]
    F --> G[LLM 호출]
    G --> H{JSON 파싱 성공?}
    H -- 예 --> I[피드백 저장]
    H -- 아니오 --> J[출력 보정 또는 재요청]
    J --> I
    I --> K[결과 반환]
```

### RAG 워크플로우

```mermaid
flowchart TD
    A[지도 가이드 문서] --> B[문서 로딩]
    B --> C[청킹]
    C --> D[임베딩 생성]
    D --> E[벡터 저장소 저장]
    F[오늘 수업 기록] --> G[검색 쿼리 생성]
    G --> H[유사 문서 검색]
    H --> I[Top 3 문서 반환]
    I --> J[프롬프트에 삽입]
```

### 예외 처리 흐름

### RAG 검색 실패

```
RAG 검색 결과 없음
→ 기본 프롬프트로 생성
→ rag_context = null 저장
```

